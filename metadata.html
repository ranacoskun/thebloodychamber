Metadata Analysis

Subsequently, we conducted an analysis of the data provided by the various institutions involved in the project, with the aim of identifying the standards and guidelines that were used to manage the data. Through this process, we discovered that most of the institutions had explicitly identified the standard to which they adhered, but some of them used several standards, therefore we suggested our choices.

For the book “The Bloody Chamber” we have chosen as a provider OPAC (Online Public Access Catalogue) which uses the standard ISBD (International Standard Bibliographic Description). It is a metadata standard which used primarily in the library community for describing bibliographic items such as books, journals, and other materials.

The provider of the manuscript, the British Library, also used ISBD, Descriptive Cataloging of Rare Materials and RDA (Resource Description and Access). However, RDA is often used in conjunction with metadata standards such as MARC 21, MODS, Dublin Core, and others to create descriptive metadata records for library resources. As the provider did not give us any explicit information about the chosen standard for photos, we decided to apply the IPTC (standard of the International Press Telecommunications Council), which is commonly used for describing news media content, such as images, videos, and text. The standard includes a variety of metadata fields, including information about the creator, copyright, location, and keywords.

The book with translation of the fairy tales is described in WorldCat, which uses combination of standards, including MARC 21, RDF and Dublin Core. MARC 21 is used to structure the bibliographic data and provide controlled access points, while Dublin Core is used to provide basic descriptive information about the resource. To be more precise, we chose MARC 21 standard to describe three our items: the audiobook, the book about exhibition in honor of Angela Carter and her autobiography, which are also provided by WorldCat.

The metadata schema used to describe the movie on IMDb (Internet Movie Database) is commonly referred as the IMDb metadata schema. Nonetheless, since we also found references to schema.org classes for the description of metadata, we chose Schema for the alignment, since it provides a wider range of classes and properites.

The magazine is held by the Jisc Library Hub Discover, which has not declared what kind of metadata standard was used. However, we found information that Jisc uses aggregation of UK academic, national and specialist library metadata, data from the Library of Congress, and International Standard Serial Numbers (ISSN), therefore we assumed that they used MARC 21, because it is one of the most widespread in UK libraries and also provided by the Library of Congress. Given that the provider does not declare explicitly any information about used standard, we chose Schema.org, because it provides a flexible and extensible vocabulary for describing various types of entities on the web, including creative works such as magazines. The standard includes vocabulary for describing information about magazines, such as their title, publisher, publication date, genre, and contributors. Using this standard, we could increase the interoperability with other entities of our project.

Information about Japan were provided by GeoNames, which uses GeoNames RDF vocabulary, a standardized system of terms to represent geographic data in a machine-readable format according to the Resource Description Framework standards (RDF). Given the specificity of GeoNames RDF vocabulary, we chose as a standard ISOinstead, because it has a wider variety of terms and therefore it's better suited for the alignment phase.

Metadata Alignment

Once the metadata standards were identified, the following step was the alignment. The purpose of this operation is to observe, by analyzing each standard's controlled terms, how different standards define the same properties, specifically the ones that were used in the conceptual map. This operation is crucial in order to better understand the meaning of the different terms and to be able to communicate it properly, ultimately allowing semantic interoperability in a potential conversation among organizations. The properties described were divided into four categories, based on the question they were answering (What? Where? When? Who?) and, therefore, the kind of entity they referred to (objects and concepts, places, dates and time-spans, people). The standards involved in the alignment are the ones defined during the metadata analysis; DC and DCTerms were also considered, since they provide the fundamental categories in the semantic description of entities.
